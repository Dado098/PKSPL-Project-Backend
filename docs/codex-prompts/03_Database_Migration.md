# 03 — Database Migrations

Convert the SQL schema below into Laravel 12 migrations. Read `00_Architecture.md` first and inspect current migrations/models. The SQL is the source of truth. Create **one migration per table**, in foreign-key dependency order. Do not merge tables, add tables, add columns, or change enum values.

## Blocking decision

The SQL primary keys are `INT PRIMARY KEY` and do not state `AUTO_INCREMENT`. Before writing migrations, ask for confirmation whether each ID is manually assigned or generated by MySQL. If the user confirms auto-generated IDs, use a consistent Laravel-compatible primary/foreign key strategy; if not, preserve manually supplied IDs and validate them in request DTOs. Do not make this decision yourself.

## Migration rules

- Use the exact table and column names from SQL.
- Use `foreignId(...)->constrained(...)` only when its unsigned-big-integer type matches the selected primary key strategy. For a strict `INT` strategy, use matching unsigned integer foreign keys with an explicit foreign constraint instead; never create mismatched FK types merely to satisfy a style rule.
- Add indexes for foreign key columns. Add only additional indexes justified by actual lookup/query requirements.
- Use `timestamps()` only for tables that define both `created_at` and `updated_at`. For tables defining only `created_at`, use only that column.
- Do not add soft deletes: the supplied SQL has no `deleted_at` fields.
- Preserve nullability exactly: no `nullable()` unless SQL permits it or the user explicitly approves it.
- Map `YEAR` intentionally and document the choice; preserve all decimal precision and scale exactly.
- Specify foreign-key delete/update behavior only after it is provided; Laravel's default restrictive behavior is preferable to guessing cascade semantics.

## SQL schema

```sql
CREATE TABLE roles (
    id_role INT PRIMARY KEY,
    nama_role VARCHAR(50),
    deskripsi TEXT
);

CREATE TABLE users (
    id_user INT PRIMARY KEY,
    id_role INT,
    nama VARCHAR(100),
    email VARCHAR(100),
    password VARCHAR(255),
    google_id VARCHAR(255),
    foto VARCHAR(255),
    status ENUM('Aktif','Nonaktif'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_role) REFERENCES roles(id_role)
);

CREATE TABLE proyek (
    id_proyek INT PRIMARY KEY,
    id_user INT,
    nama_proyek VARCHAR(150),
    tujuan_valuasi TEXT,
    lokasi VARCHAR(255),
    tahun YEAR,
    deskripsi TEXT,
    status ENUM('Draft','Proses','Selesai','Dibatalkan'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_user) REFERENCES users(id_user)
);

CREATE TABLE ekosistem (
    id_ekosistem INT PRIMARY KEY,
    nama_ekosistem VARCHAR(150),
    deskripsi TEXT,
    status ENUM('Aktif','Nonaktif'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE area_terdampak (
    id_area INT PRIMARY KEY,
    id_proyek INT,
    id_ekosistem INT,
    nama_area VARCHAR(150),
    latitude DECIMAL(10,6),
    longitude DECIMAL(10,6),
    luas DECIMAL(15,2),
    satuan_luas VARCHAR(20),
    deskripsi TEXT,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_proyek) REFERENCES proyek(id_proyek),
    FOREIGN KEY (id_ekosistem) REFERENCES ekosistem(id_ekosistem)
);

CREATE TABLE provisioning_service (
    id_provisioning INT PRIMARY KEY,
    id_area INT,
    nama_objek VARCHAR(150),
    produktivitas DECIMAL(15,4),
    harga_pasar DECIMAL(15,2),
    luas_pemanfaatan DECIMAL(15,2),
    satuan_luas VARCHAR(20),
    referensi TEXT,
    nilai DECIMAL(18,2),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_area) REFERENCES area_terdampak(id_area)
);

CREATE TABLE regulating_service (
    id_regulating INT PRIMARY KEY,
    id_area INT,
    jenis_regulating VARCHAR(150),
    indikator VARCHAR(150),
    satuan VARCHAR(50),
    nilai_indikator DECIMAL(15,4),
    referensi TEXT,
    nilai DECIMAL(18,2),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_area) REFERENCES area_terdampak(id_area)
);

CREATE TABLE supporting_service (
    id_supporting INT PRIMARY KEY,
    id_area INT,
    fungsi_pendukung VARCHAR(150),
    deskripsi TEXT,
    referensi TEXT,
    nilai DECIMAL(18,2),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_area) REFERENCES area_terdampak(id_area)
);

CREATE TABLE cultural_service (
    id_cultural INT PRIMARY KEY,
    id_area INT,
    nama_aktivitas VARCHAR(150),
    jumlah_pengunjung DECIMAL(15,2),
    biaya_perjalanan DECIMAL(15,2),
    frekuensi DECIMAL(15,2),
    referensi TEXT,
    nilai DECIMAL(18,2),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_area) REFERENCES area_terdampak(id_area)
);

CREATE TABLE metode_valuasi (
    id_metode INT PRIMARY KEY,
    nama_metode VARCHAR(150),
    deskripsi TEXT,
    formula TEXT,
    parameter TEXT,
    status ENUM('Aktif','Nonaktif'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE hasil_valuasi (
    id_hasil INT PRIMARY KEY,
    id_area INT,
    id_metode INT,
    direct_use_value DECIMAL(18,2),
    indirect_use_value DECIMAL(18,2),
    option_value DECIMAL(18,2),
    existence_value DECIMAL(18,2),
    bequest_value DECIMAL(18,2),
    tev DECIMAL(18,2),
    tanggal_hitung DATETIME,
    keterangan TEXT,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_area) REFERENCES area_terdampak(id_area),
    FOREIGN KEY (id_metode) REFERENCES metode_valuasi(id_metode)
);

CREATE TABLE dataset_referensi (
    id_dataset INT PRIMARY KEY,
    nama_dataset VARCHAR(150),
    kategori VARCHAR(100),
    tahun YEAR,
    file_dataset VARCHAR(255),
    sumber VARCHAR(255),
    deskripsi TEXT,
    status ENUM('Aktif','Nonaktif'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE basis_data_ai (
    id_basis INT PRIMARY KEY,
    nama_basis VARCHAR(150),
    versi VARCHAR(100),
    deskripsi TEXT,
    path_file VARCHAR(255),
    status ENUM('Aktif','Nonaktif'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE analisis_ai (
    id_analisis INT PRIMARY KEY,
    id_proyek INT,
    id_user INT,
    pertanyaan TEXT,
    jawaban TEXT,
    sumber_data TEXT,
    created_at TIMESTAMP,
    FOREIGN KEY (id_proyek) REFERENCES proyek(id_proyek),
    FOREIGN KEY (id_user) REFERENCES users(id_user)
);

CREATE TABLE histori (
    id_histori INT PRIMARY KEY,
    id_user INT,
    id_proyek INT,
    aktivitas VARCHAR(255),
    keterangan TEXT,
    created_at TIMESTAMP,
    FOREIGN KEY (id_user) REFERENCES users(id_user),
    FOREIGN KEY (id_proyek) REFERENCES proyek(id_proyek)
);

CREATE TABLE validasi_analyst (
    id_validasi INT PRIMARY KEY,
    id_hasil INT,
    id_user INT,
    status_validasi ENUM('Valid','Revisi','Ditolak'),
    metode_analisis ENUM('Manual','AI'),
    catatan TEXT,
    tanggal_validasi DATETIME,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (id_hasil) REFERENCES hasil_valuasi(id_hasil),
    FOREIGN KEY (id_user) REFERENCES users(id_user)
);
```

## Verification

After the key strategy is confirmed, run migrations in the Docker `app` container and use `php artisan migrate:status`. Add a focused migration test only if the project has an established migration-test pattern.
